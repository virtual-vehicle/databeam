Description

A module to receive data over a TCP connection. Allows for bytestream or json formatted data and can be set to receive as a server or a client.

Configuration

  • TCP Address: The TCP address to use. In server mode it defines the address to listen on, in client mode the address to connect to.
  • TCP Port: The TCP port to use with the address.
  • Client Mode: Can be set to put the module in client mode, where it creates a TCP client and connects to a TCP server. Otherwise, it creates a TCP server where one client can connect and send its data to.
  • Use Length Bytes: Toggle this, if the data sender sends the length of each data packet before the data itself. Allows for more robust data transfer.
  • Number Length Bytes: If a length byte is used, this field defines how many bytes the length uses.
  • Big Endian: If set, all data and the length is interpreted in big-endian.
  • Data Format: Defines the type of data which is sent to the module.
    • struct: Expects a byte array. Please use the "Struct Entries" field to define what type of data is expected.
    • json: Expects a json-formatted data packet. This setting ignores the "Struct Entries" field.
  • Struct Entries: If "Data format" is set to struct, this field defines the data which is expected.

Known Issues

In server mode, when a measurement is stopped, it closes the connection to directly listen to a new one. This must be considered when connecting a data sender client.
